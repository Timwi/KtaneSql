On the subject of SQL

SQL is a programming language that describes how to manipulate sources of data and generate a difference output source. It is comprised of many different keywords, only a minimal list is used in this module.

How SQL is processed

SQL is processed always in the same way:

  1. Each row is compared against the filters. If a row complies with the filter expression it is kept for further processing. (See WHERE clauses below for more information)
  2. Each row that passes the filter expression is then processed against the selection expressions. (See SELECT clauses below for more information)
  3. Finaly, each row of the end result is processed against the LIMIT clause; that is SKIP X amount of rows from the result and then TAKE only Y lines after that. (See LIMIT clauses below for more information)

When you SELECT something, you choose to take all the values that match this X, Y or Z expression and return it. WHERE X operation Y

WHERE introduces filters and all rows will be tested against the conditions you provide the module with. A row is kept in the output only if it complies with the filter condition. This SQL emulator makes it simple for you, you can only compare columns to fixed values such as: A < 2, C = 3, F >= 5. The possible operators are all simple mathematical operators that you already know (In most cases) such as:

  • Equals or "="
  • Not equals or "<>"
  • Less than or "<"
  • Less than or equal or "<="
  • Greather than or ">"
  • Greather than or equal or ">="

Here's a basic example of a single filter:

LIMIT X, Y

LIMIT comes with one or two values. For simplicity, the SQL module splits them for you in two selectors:

  • LIMIT: Indicates the number of rows to keep from the matching results after applying conditions. If you select "All", no rows are discarded.
  • SKIP: Indicates the number of rows to skip after applying conditions. If you select "None", no rows are discarded.
